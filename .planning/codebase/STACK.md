# Technology Stack

**Analysis Date:** 2026-03-17

## Languages

**Primary:**
- TypeScript 5.7.2 (`package.json`) - `src/main.ts` and `src/shared/CameraMath.ts` implement the canvas-based runtime logic that compiles to ES2020 modules.

**Secondary:**
- HTML/CSS (`index.html`, `editor/index.html`, `editor/styles.css`) - shell markup and styling for the runtime container and the editor UI that the TypeScript runtime manipulates.

## Runtime

**Environment:**
- Modern browsers (ES2020/ESNext modules as defined in `tsconfig.json`) host the runtime; `src/main.ts` renders into `<canvas id="vk-canvas">` inside `index.html` and fetches `/assets/maps/active-map.json`.
- Node.js drives the Vite dev server and build binaries invoked by `npm run dev` / `npm run build` in `package.json`, which load `vite/config.dev.mjs` and `vite/config.prod.mjs`.

**Package Manager:**
- `npm` (use `npm install`; `package-lock.json` locks dependency versions).

## Frameworks

**Core:**
- Vite 6.3.1 (`package.json`) – dev server/bundler defined via `vite/config.dev.mjs`/`vite/config.prod.mjs`, including the custom `editorAssetsPlugin`.
- Custom `editorAssetsPlugin` (`editor/vite/editorAssetsPlugin.mjs`) – enumerates `public/assets/*`, exposes `/__*` middleware, and mirrors `public/assets/maps` into `dist/assets/maps`.

**Testing:**
- Not detected (no test scripts or testing dependencies in `package.json`).

**Build/Dev:**
- Terser 5.39.0 (`package.json`) – `vite/config.prod.mjs` sets `minify: 'terser'` so production builds shrink without additional tooling.

## Key Dependencies

**Critical:**
- TypeScript 5.7.2 (`package.json`) – compiler for the runtime and helper modules with `tsconfig.json` strict/no-unused settings.
- Vite 6.3.1 (`package.json`) – orchestrates canvasing bundling, dev server, and the editor entry point; configured in `vite/config.dev.mjs`/`vite/config.prod.mjs`.
- Terser 5.39.0 (`package.json`) – production minifier wired through `vite/config.prod.mjs`.

**Infrastructure:**
- Node core `fs` and `path` (used in `editor/vite/editorAssetsPlugin.mjs`) – scan asset directories and copy `public/assets/maps` to `dist/assets/maps`.

## Configuration

**Environment:**
- `tsconfig.json` targets `ES2020`, `module: "ESNext"`, `moduleResolution: "bundler"`, and includes `src` while enabling `strict`, `noUnusedLocals`, `noUnusedParameters`, and `skipLibCheck`.

**Build:**
- `vite/config.dev.mjs` (base `./`, dev server port 5173, multi-entry points `index.html` & `editor/index.html`, `editorAssetsPlugin` plugin).
- `vite/config.prod.mjs` (produces `dist`, minifies via Terser, reuses the same rollup input, and relies on `editorAssetsPlugin` for asset-aware middleware).

## Platform Requirements

**Development:**
- Node.js + npm to run `npm run dev` (`vite --config vite/config.dev.mjs`) and `npm run build` (`vite build --config vite/config.prod.mjs`); these commands also trigger `editor/vite/editorAssetsPlugin.mjs` to watch `public/assets/*` and expose local `/__*` endpoints.

**Production:**
- Static hosting for `dist/` (generated by `vite/config.prod.mjs`); `editorAssetsPlugin` mirrors map JSON into `dist/assets/maps` so the runtime in `src/main.ts` continues to load `/assets/maps/active-map.json`.
