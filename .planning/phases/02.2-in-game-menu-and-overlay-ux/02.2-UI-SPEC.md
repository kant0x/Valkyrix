# Phase 02.2 - UI Design Contract

> Visual and interaction contract for the in-match command layer, ESC system overlay, and end-of-match overlay family.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none |
| Icon library | none |
| Font | `"Trebuchet MS", "Segoe UI", sans-serif` for body/labels; `Georgia, "Times New Roman", serif` for headings/display |

Manual design system declared: this phase uses vanilla TypeScript + DOM modules with local style injection and phase-specific CSS tokens.

---

## Screen Inventory

| Surface | Role | Priority |
|---------|------|----------|
| Bottom command bar | Primary in-match battle workspace | Primary focal point |
| ESC system overlay | Short live-match system actions | Secondary |
| Victory/defeat overlay | End-of-match decisive state | Tertiary but dominant when active |

Primary focal point: the center-left build/action zone of the bottom command bar.

Visual hierarchy:
1. Build/command actions
2. Current resources and selected action state
3. Wave/citadel status
4. ESC system actions

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline separators, icon gaps |
| sm | 8px | Compact chip padding, micro spacing |
| md | 16px | Default inner spacing for command modules |
| lg | 24px | Section padding inside the bar and overlays |
| xl | 32px | Gaps between major control groups |
| 2xl | 48px | Overlay internal breathing room |
| 3xl | 64px | Full-screen result overlay composition spacing |

Exceptions: none

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 500 | 1.5 |
| Label | 14px | 700 | 1.2 |
| Heading | 20px | 700 | 1.2 |
| Display | 28px | 700 | 1.1 |

Type scale rule:
- Labels communicate tactical metadata
- Body handles readable system and action text
- Heading is used for section heads and overlay titles
- Display is reserved for result states and command-surface anchors only

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#07111d` | Match background, command bar base, overlay field |
| Secondary (30%) | `#14263a` | Cards, segmented trays, raised surfaces, ESC panel |
| Accent (10%) | `#84c7ff` | Primary command highlight, active build state, focus ring, selected command chip, rune glow trims |
| Destructive | `#c45b5b` | Exit confirmation, defeat-state destructive controls only |

Accent reserved for:
- Selected build action
- Primary in-match command CTA
- Active focus ring
- Current tactical state highlight
- Key rune-line separators

Accent is NOT used for every interactive element. Neutral actions remain secondary surfaces.

---

## Component Contract

| Region | Content | Interaction Rule |
|--------|---------|------------------|
| Build tray | Attack tower, buff tower, staged unit slot(s) | Only one build/train intent can be armed at a time |
| Resource strip | Energy/resources and current affordability cues | Must always remain visible |
| Battle status | Wave, citadel HP, live readiness | Read-only, never mixed with primary CTA styling |
| System edge | ESC hint and secondary utilities | Never outranks build tray visually |

Command bar behavior:
- Always visible during active match
- Reads as a single bottom battle console, not floating isolated buttons
- Uses grouped trays/cards instead of loose elements
- Selected action must have a persistent visible armed state until canceled or consumed

ESC overlay behavior:
- Compact system overlay only
- No gameplay build actions inside ESC
- Resume, music/settings, and exit flow only
- Match simulation continues underneath

Result overlay behavior:
- Same visual family as command layer
- Stronger contrast and display typography
- Single dominant result CTA

---

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Arm Attack Tower` |
| Empty state heading | `Command Slot Locked` |
| Empty state body | `This war control is staged for a later step. Hold the line with towers and resources for now.` |
| Error state | `Action unavailable - not enough resources or invalid placement. Rebuild your economy or choose another tile.` |
| Destructive confirmation | `Exit to Menu`: `Progress from this battle will be lost. Leave the field?` |

Secondary action copy:
- Buff tower action: `Arm Buff Tower`
- Resume action: `Return to Battle`
- Music/settings area: short labels only, no lore-heavy verbosity

Icon-only controls:
- Not allowed in first implementation without text labels
- Any compact control must still expose visible label text

---

## Motion Contract

- Command bar should feel stable and live, not cinematic
- Hover/selection transitions use short 150-180ms ease changes
- Armed command state may glow softly but must not pulse aggressively
- ESC overlay enters with a soft fade, not a hard pop
- Result overlay can use stronger emphasis, but remains readable and immediate

---

## Accessibility and State Rules

- Primary command state must be visually distinguishable by both color and border/shape treatment
- Error state must be shown inline in the command layer or system overlay with clear recovery path
- Destructive actions must use destructive color plus confirmation copy
- Text labels remain visible on all command buttons
- Bottom command bar must remain usable on narrower desktop widths without hiding critical resources
